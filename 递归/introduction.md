# 递归
### 什么是递归
递归，类似于数学数列中的递推公式，就是函数调用自己。<br>
### 它可以用来解决什么问题
一般通过循环能解决的问题递归都能搞定。除此之外，递归可以很好地处理一些倒序问题。<br>
### 优缺点
1. **优点**
可以完全复刻数学中的递推公式，简洁的描述代码与背后蕴含的数学关系。<br>
如斐波那契数列， $f(x) = f(x-1)+f(x-2) $ , 递归可以完全按照数学表达式的逻辑来编写。<br>
**更重要的是递归可以很好的处理倒序问题**,如我们要将某个十进制数转化为二进制数，如将5转化为二进制为101,5%2=1(就是将它的二进制的末尾保留了下来),5/2=2（就是将它二进制的前二位保留了下来,这里默认使用int类型），如果此时直接输出5%2，就直接将5的二进制末尾先打印了出来，不符合要求。其实我们只需要用一个递归，将打印放在函数调用与return语句之间就可以实现倒序打印<br>如下列程序<br>
```
int to_base_n(int num,int dig)
{
	int rest;
	
	if(dig <2 || dig >10)
	{
		return -1;
	}else if(num < 0)
	{
		putchar('-');
		to_base_n(-num,dig);//点睛之笔，不在当前帧修改num，而将num的变化值交给下一帧 
		return 0;
	}
	rest = num % dig;
	if(num >= dig)//终止条件变量，比如这里的num，在递归函数的函数体中不能变化，这样才能保持递归《最贴近数学函数》的本色。 
		to_base_n(num/dig,dig);//仅在再次调用时变化 
	putchar('0'+rest);//putchar的高端应用! 
	return 0;
}
```
`putchar`语句位于调用与`return`之间，这样就可以实现倒序打印。<br>
同时本程序也很好的使用了`putchar`;以及表达了递归的本质：数学中的递推公式。也就是说终止条件变量仅在调用函数时变化,而将变化后的值传递给递归函数的下一级形参，如这个程序的 `to_base_n(num/dig,dig);`中的`num`作为终止条件变量，仅在再次调用时变化，而符合终止条件时终止递归。<br>
就像数学表达式 $f(x)=f([x/2])$ 所描述的那样,层层递推，直到其定义域时终止。<br>
2. **缺点**
递归的缺点就是占内存。每一个递归函数在栈中有空间，当递归过多，栈容易溢出。<br>
3. **总结**
需要倒序的场景，可以考虑递归。否则慎用。<br>
### 注意的地方
递归的终止条件必不可少，与其相关的变量要在再次调用函数时，将这个变量变化后的值传给下一级函数，这样就保证了递归发挥了其简明地表达数学关系的优点 <br>
```
double power(double base,int pow)
{
	double num = base;
	bool enfor = true;
	
	if(0.0==base && pow<= 0)
	{
		printf("err\n");
		num = 0.0;
		enfor = false;
	}else if(pow < 0)
	{
		num = power(1/base,-pow);
	}else if(0==pow)
	{
		num = 1.0;
	}
	if(enfor&&pow>1)
	{
		num = base*power(base,pow-1);
	}
	return num;
} 
```
`pow`仅在再次调用函数时将其变化后的值传给下级函数，保证了每一级函数的`pow`在本级都是相同的,没有出现被赋值，或被增减的情况，大大增加了代码的稳定性和可读性，也简明表达了 $f(1)=b,f(0)=1，当p>1时，f(p)=bf(p-1)，当p<0时, f(p)=f(-p)$ 的数学表达式。
